## Code Rules

The generated code must follow ALL of these rules strictly:

1. **No imports** — Do not write any import statement under any circumstances.

2. **One class only** — Define exactly one class that inherits from `CoreStrategy`. No helper classes, no standalone functions outside the class.

3. **No comments** — Do not include any inline comments or docstrings anywhere in the code.

4. **Method signatures** — Override the relevant methods from the base class:
   - `calculate_indicators(self, data: pd.DataFrame) -> pd.DataFrame` — adds computed indicator columns to the dataframe and returns it
   - `long_entry_signals(self, data: pd.DataFrame) -> pd.DataFrame` — adds a `long_entry` column (1 = enter long, 0 = no signal) and returns it
   - `short_entry_signals(self, data: pd.DataFrame) -> pd.DataFrame` — adds a `short_entry` column (1 = enter short, 0 = no signal) and returns it
   - `long_exit_signals(self, data: pd.DataFrame) -> pd.DataFrame` — adds a `long_exit` column (1 = exit long, 0 = no signal) and returns it
   - `short_exit_signals(self, data: pd.DataFrame) -> pd.DataFrame` — adds a `short_exit` column (1 = exit short, 0 = no signal) and returns it
   - `position_size(self, balance: float, entry_price: float) -> float` — returns the position size as a float (abstract, MUST be implemented)

5. **Signal columns** — Signal columns must contain integer values 1 or 0 only. Always chain `.fillna(0).astype(int)` when assigning signal columns.

6. **No arbitrary libraries** — Only `pd` and `np` are available. Never use ta, sklearn, scipy, or any other library.

7. **Safe dataframe operations** — Always use `.loc[]` for column assignments. Handle NaN values with `.fillna(0)` before casting to int.

8. **Class naming** — Use a descriptive PascalCase name that reflects the strategy (e.g. `RSIMeanReversionStrategy`, `BollingerBandBreakoutStrategy`).

9. **Position sizing default** — If the user does not specify a risk management or position sizing rule, default to: `return balance / entry_price`
